# Documentation Templates

Use these templates when standardizing a repository README under the Tanaab documentation rules.

## Available Templates

- `readme-full/README.md`: use when the repository can keep all durable user-facing documentation in one README.
- `readme-docs-wrapper/README.md`: use when the README should stay lightweight and delegate durable docs to a VitePress docs site.

## Choosing a Template

- Start with the full README template by default.
- Switch to the docs wrapper template when the repository needs multiple durable guides, large reference sections, or separate navigation layers.
- When using the docs wrapper template, assume the fuller docs surface will be VitePress unless the repository already has another approved docs stack.

For the detailed decision rule, see [../../../tanaab-documentation/references/readme-standards.md](../../../tanaab-documentation/references/readme-standards.md).
